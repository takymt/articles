## はじめに

2026年3月初頭に、カーネギーメロン大学の [Master of Software Engineering (MSE)](https://mse.s3d.cmu.edu/applicants/mse-as/index.html) というプログラムから合格通知をいただき、今年の8月からアメリカに行くことになりました。

本記事では、米国大学院留学を志望するにあたってお世話になった方々への謝辞を述べるとともに、今後同様の挑戦をされる方々に向けての少しばかりの情報共有が出来ればと思います。

## お世話になった方々への謝辞

今回大学院に行くにあたり、以下の方々に大変お世話になりました。本当にありがとうございました。

- 自分の留学を誰よりも応援し支えてくれた妻
- 推薦状を快く書いてくださった上司の方々
- 事務面の多くを引き受けてくれた社内の海外留学プログラム担当の方々
- 現地の有益な情報を提供してくださった CMU 在学生、卒業生の知人の方々
- 心強いサポートをくれたエージェントの方々(留学、住居、ビザ、英語...etc.)

現地での最大限のコミットメントをもって恩返し出来ればと思います。

## 米国大学院留学を志望したきっかけ

- 非CS系学部卒であり、体系知の不足が業務の色々な所でネックになっていた
- ソフトウェアの最先端であるアメリカで修行したかった
- 非日本語圏に住んでみたかった

あたりが理由としてあります。

最初はジョージア工科大学のCS修士や、UoPeopleのCS学士といったオンライン留学を検討していたのですが、どうせなら現地で学業以上の体験をしたいと思ったのと、CS学士に関しては今更後追いで取った所で自分のアイデンティティにはならないと思い、学士をスキップして現地での修士留学を志すようになりました。

また、Computer Science ではなく Software Engineering を選んだ理由としては、

- CS修士は非常に人気で倍率が高く、CS学士がないと門前払いされることも多い（ので日和った）
- CS畑ではない分、プログラミングよりも開発手法/設計/コミュニケーション/リーダーシップみたいなもの（つまりはソフトウェア工学の領域）の方が取っつきやすいという感覚があった

といったところです。

## 準備したこと

### 大学選定 (2025/4)

自分の場合はソフトウェア工学というCS系王道ではない分野だったので、そもそもの選択肢が少なかったのですが、
そのうえで、以下の条件を満たす大学院を選んでいきました。

- プロジェクトやコースワークなど、実践的な部分に重きを追いている
- 現地キャンパスのフルタイムである
- アメリカの大学である

基本的には、自分の興味ある研究テーマが強いところか、それ以外は US News とかのランキングをみて上から順に当てはなるところを選んでいくことになると思いますが、最終的に自分は以下の6校に出願することにしました。

- 第1志望: [Carnegie Mellon Univ. MSE](https://mse.s3d.cmu.edu/applicants/mse-as/index.html)
- 第2志望: [MaryLand Univ. Software Engineering](https://academiccatalog.umd.edu/graduate/programs/software-engineering-pmes/)
- 第3志望群(滑り止め)
    - [Arizona State Univ. Software Engineering, MS](https://degrees.apps.asu.edu/masters-phd/major/ASU00/TSSERMS/software-engineering-ms)
    - [University of Arizona Software Engineering](https://online.arizona.edu/programs/graduate/online-master-science-software-engineering-ms)
- 第4志望群(滑り止め)
    - [Northeastern Univ. Software Engineering Systems](https://graduate.northeastern.edu/programs/ms-software-engineerng-systems/master-of-science-in-software-engineering-systems/)
    - [Stevens Inistitute of Tech. Software Engineering Master’s Program](https://www.stevens.edu/program/software-engineering-masters)

第3, 4志望群については、ソフトウェア工学ど真ん中という感じではなく「CSプログラムの焼き直し」だと感じたので、若干志望度が下がっています。

### TOEFLスコアメイク (2025/5-2025/9)

これが一番大変でした。特にピーク時は週に4-50時間は英語の勉強をしていて、かつ2週間に1回くらいのペースで本試験を受ける生活をしていました。

自分の場合は、目標は総合スコア100点だったのですが、8月頭に受けた2回目の試験で早々に99点を取ってしまったせいで、その後油断したのが本当に良くなかったです。以降ズルズルとスコアが1点足りない状況が続き、本当に精神衛生に毒でした。

現実逃避として、DSのどうぶつの森で夏の魚をコンプリートしたり、サッカー日本代表の遠藤選手が所属するFCリヴァプールの日本凱旋エキシビジョンマッチを見に行ったりしてました。ファン・ダイクが遠藤にキャプテンマーク渡すとこマジでアツすぎた。

[AGOSジャパン](https://www.agos.co.jp/)という語学学校に4か月くらい通いつつ、最終的には合計スコア100点、MyBest 106点 (R30/L27/S20/W29) を達成することができたので、本当にホッとしました。スピーキングはもうちょっと何とかしたかった感がありますが、もう正直二度と受けたくないです。

### GRE (2025/9-2025/10)

ネイティブ用のセンター試験みたいなものらしいです。

TOEFL と比べて受験される方が少ないと思うので詳細は割愛しますが、見たこともない単語を1500語くらい覚えればそこそこの点が取れるので、再現性という意味ではTOEFLより圧倒的に楽です（楽とは言ってない）。

さらに数学に限って言えば、日本の中学代数幾何レベルの問題が大半なので、理系修士に行かれるような方であれば基本的に満点近い点数が狙えます。日本の教育水準に感謝。

制度上、一定期間内受験できる回数が決まっているので、そこだけご注意下さい。

### 出願書類準備 (2025/7-2025/12)

志望動機、履歴書、推薦状の他、ビデオエッセイ等をケースバイケースで容易することになります。

#### 志望動機 (Statement of Purpose)

社内の留学プログラム申請時に一通り棚卸し等を実施していたので、割とサクサク書くことが出来ました。

逆に反省としては、

- 英語スコアメイクにかまけてないで、もっとSoPに時間をかければよかった。
- 留学コンサルの方だけではなく、社内の方など専門家にももっとたくさん見てもらえばよかった。

あたりがあります。

もちろん留学コンサルのネイティブの方に見てもらうのも重要なのですが、非ネイティブの英語文法にはそこまで重きが置かれてない雰囲気があるというか、中身のほうが圧倒的に重要視される感があります。なので、もし時間がなければ、最悪grammarlyとAIチェッカーのみで提出しちゃってもOKだと思います。

余談ですが、自分はずっとライティングの勉強にChatGPTを使っていたせいか、GPT系の癖がうつってしまい、フル手書きでもAIチェッカーにそこそこ引っかかるのが地味に大変でした。同様の勉強法をされる場合は、手書きだったとしても自己防衛としてAIチェッカーはかけておくと良いのかなと思います。

#### 履歴書 (Resume/CV)

自分の場合は非CS学士であるため、あまりアカデミックなことは書けず、勤務先や個人事業主としての活動を淡々と書きました。自信を持って書ける実績がそこまでなく、体外的なアウトプットをもっとしないといけないなと思ったのを覚えています。

#### 推薦状 (Recommendation Letter)

自分は大学時代ギターとゲームばっかりしてたので、当時の教授ではなく、過去の上司御三方に推薦状を依頼しました。
今思えば、変にバランスを考えて面識の浅い当時の教授にお願いするよりも、自分をよく知る身近な方々にお願いしてよかったと感じています。

#### その他 (ビデオエッセイなど)

大学によってケースバイケースですが、カーネギーメロンの場合は「自分が熱中しているものについて動画で教えて」という内容だったので、大学から続けているギターの話をしました。

インパクトを狙ってギター演奏シーンを入れつつ「ギター最高！キャリアにも活きてる！」みたいな話をしたらちょっとウケたっぽく、合格通知でも興味深かった点として言及されていました。「スベってたはいたけど言及しやすかっただけ説」も(ry

### 出願条件 (Prerequisite)

カーネギーメロン含め多くの出願校で「CS系の学士を取得していること」が要件にあったのですが、自分の場合はこれを満たせないことが確定していたので、以下の回避策を取りました。

- ピュアなCS系には出願せず、実務思考の強そうなソフトウェア工学修士を選ぶ
- 出願前に「出願しても良い？」的なメールをResumeとともに送り、ウェルカムな反応だったら出願する
- ResumeやSoPでスキルを匂わせる
- GREとコーディング試験で数学的/CS的素養を匂わせる

似た境遇の方はそこまでいらっしゃらないかもですが、参考に慣ればと思います。

### コーディング試験

カーネギーメロン大学のみコーディング試験がありました。

こちらについては情報がほとんど開示されていなかったため、[LeetCode](https://leetcode.com/) という競技プログラミングの Medium-Hard をひたすら解いて対策しました。

もちろん内容には言及できませんが、この期間でデータ構造とかアルゴリズムとか色々勉強になりました。

## 結果

ソリッドなCS経歴を持っていればもっと余裕があったのかもなとは思います。

- [Carnegie Mellon Univ. MSE](https://mse.s3d.cmu.edu/applicants/mse-as/index.html): 合格
- [MaryLand Univ. Software Engineering](https://academiccatalog.umd.edu/graduate/programs/software-engineering-pmes/): 不合格
- [Arizona State Univ. Software Engineering, MS](https://degrees.apps.asu.edu/masters-phd/major/ASU00/TSSERMS/software-engineering-ms): 不合格
- [University of Arizona Software Engineering](https://online.arizona.edu/programs/graduate/online-master-science-software-engineering-ms): 合格
- [Northeastern Univ. Software Engineering Systems](https://graduate.northeastern.edu/programs/ms-software-engineerng-systems/master-of-science-in-software-engineering-systems/): CMU合格のため見送り
- [Stevens Inistitute of Tech. Software Engineering Master’s Program](https://www.stevens.edu/program/software-engineering-masters): CMU合格のため見送り

## 今後の抱負

AIでソフトウェアエンジニアの職が無くなると言われている時代に、あえてクラシックな勉強をしにいくことには正直不安があります。ただもしみんな職を失うなら、まだ逆張る方が生存確率が高そうな気もするので、気合い入れて頑張ってこようと思います。学業以外のコミュニケーションとかリーダーシップみたいなソフトスキルも成長させたい。

ちなみに最近は、5月ごろから開始されている [Introduction to Computer Systems](https://www.cs.cmu.edu/~213/) という授業を受けていて、AI禁止のストロングな環境で、アセンブリを読んでBOFをしたりキャッシュ戦略を最適化したりしてます。拘束時間的にもまあまあヘビーで、「全米で一番徹夜率が高い大学」という眉唾な噂も伊達ではないのですが、それでも今まで知らなかったことがどんどん分かってくる感覚は何者にも代えがたいなと思います。

現地にいってからも今の2-3倍は大変になるはずで、もちろん不安もあるのですが、やっぱり期待が大きいなと思います。
日本大好き怠惰人間の自分がこんなことになるとは本当に人生何があるかわからんのですが、人生で二度あるかわからない機会なので、全力で楽しんで胸を張って日本に返ってきたいなと思います。

おれさ...日本帰ったら、絶対ぽこポケするんだ...！

ではでは！
